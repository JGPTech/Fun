# ERDŐS #604 STEAMROLLER: OEIS A160663 Extension

A high-performance computational framework for enumerating distinct squared distances generated by lattice points in finite square regions. This project extends the OEIS sequence [A160663](https://oeis.org/A160663) by a factor of 30, computing exact terms up to $m = 300{,}000$.

## Overview

Let $D(m)$ be the number of distinct squared Euclidean distances from the origin to points in the integer lattice box $[0,m]\times[0,m]$. This repository contains the code used to generate the exact values of $D(m)$, culminating in a 20+ GB in-memory bitset sweep that produced an extended b-file for $m \in [1, 300{,}000]$.

Alongside the exact enumeration, this project empirically analyzes the Landau–Ramanujan-motivated normalized growth statistic:

$$c_n = \frac{D(m)}{n / \sqrt{\log n}}, \qquad n = (2m)^2$$

Over the computed range, $c_n$ exhibits sustained empirical stabilization near **0.28928**.

## Architecture

The project consists of two structurally independent components, providing high-confidence cross-validation of the generated sequence:

### 1. The Steamroller (`01_steamroller.py`)

The primary generator. It utilizes a cumulative layer-based decomposition:

$$S_k = \{ k^2 + j^2 : 0 \le j \le k \}$$

By sweeping through $k = 1 \dots M_{max}$, it guarantees no redundant distance checks.

* **Backend:** Numba JIT-compiled Python.
* **Memory Management:** Operates on a raw, flat 1-dimensional `numpy.uint8` bitset array. Uses deterministic, safe bitwise updates to avoid allocation overhead and mutation race conditions.
* **Output:** Generates `b160663.txt` (OEIS standard format) and a rich CSV of $c_n$ statistics.

### 2. The Verifier (`02_verify_a160663.py`)

A structurally independent validation suite. Cross-checking a 21 GB bitset requires algorithmic redundancy, not just re-running the same code. The verifier utilizes two alternate paradigms:

* **Method A (Direct Enumeration):** Iterates over values $v \le 2m^2$ and uses a custom exact-integer Newton's method square root (`_isqrt`) to verify sum-of-two-squares representations. Completely bypasses the bitset layer logic.
* **Method B (Alternative Sieve):** Uses a boolean flat-array sieve with a differing $a \le b$ loop structure to independently mark and tally distances.

A self-test first confirms that Methods A and B agree on small inputs, validating the verifier itself before it is applied to the b-file.

## Hardware Requirements

To run the steamroller to the full $M_{max} = 300{,}000$, your machine must have substantial memory bandwidth and capacity.

* **RAM:** Minimum 32 GB (the $m = 300{,}000$ bitset alone requires exactly **20.95 GiB** of continuous allocation).
* **Compute:** Any modern multi-core CPU. The Numba kernel optimizes heavily for vectorization.

*(Note: You can easily run smaller bounds on standard hardware by modifying `M_MAX` in `01_steamroller.py`.)*

## Usage

**1. Generate the Sequence**

```bash
python 01_steamroller.py
```

*Outputs: `b160663.txt` and `erdos_604_full_stats.csv`.*

**2. Run the Independent Verifier**

```bash
python 02_verify_a160663.py b160663.txt
```

*Outputs: Pass/Fail matrix comparing the b-file against direct enumeration and sieve models at critical sample points.*
